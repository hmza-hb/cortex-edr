# Troubleshooting & Support

Everything doesn't always go according to protocol. This guide helps you resolve common technical obstacles with the CortexEDR platform.

## Common Issues

### 1. Repository Clone Failures
If Cortex is unable to clone your repository, verify the following:
- **Public URL**: Ensure the URL is accessible without authentication (for non-connected accounts).
- **Private Access**: For private repos, ensure you have authorized the **Cortex GitHub App**.
- **Large Repositories**: Repositories over 500MB may timeout during the initial clone phase.

### 2. AI Synthesis Timeouts
The synthesis phase (Agent 6) can be computationally intense for massive monorepos.
- **Solution**: Upgrade to the **Developer** or **Teams** tier for dedicated reasoning compute and higher timeout thresholds.
- **Selective Scans**: Use `.cortex-ignore` (coming soon) to skip non-critical directories.

### 3. Billing & Tier Sync
If you've upgraded but don't see your new limits:
1.  Verify the payment was successful in [Paddle](https://paddle.com).
2.  Refresh the dashboard to trigger a profile sync.
3.  If issues persist, contact support with your **Order ID**.
