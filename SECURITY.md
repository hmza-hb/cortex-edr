# Security Policy

## Supported Versions

We take the security of Cortex very seriously. The following versions of Cortex are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| v2.0.x  | :white_check_mark: |
| v1.x    | :x:                |

## Reporting a Vulnerability

If you find a security vulnerability or bug in Cortex, please do **not** open a public GitHub issue. Instead, report it privately to our security team.

### Reporting Process

1. Email your report to **cortexedr@gmail.com**.
2. Include the following details in your report:
   - Type of issue (e.g., SQL injection, XSS, privilege escalation, secret leak).
   - Component affected (e.g., Landing Site, EDR Dashboard, CLI module).
   - Detailed steps to reproduce the issue (including proof of concept scripts or requests).
   - Potential impact of the exploit.

### Response SLA

Our security response SLA commitments:
- **Acknowledgment**: Within 48 hours of initial report receipt.
- **Triage & Status**: Updates every 7 days until resolution.
- **Resolution target**: Security patches deployed within 14 days of confirmation.

We ask you to follow responsible disclosure guidelines and allow us time to release security updates before disclosing vulnerabilities publicly. Thank you for keeping Cortex secure!
